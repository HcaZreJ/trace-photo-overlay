# 轨迹照片九宫格 · trace-photo-overlay

把运动轨迹叠加到九宫格照片上，一键导出朋友圈九图。骑行、徒步、越野跑、跑步、滑雪……任何 GPS 轨迹都能用。

纯前端、零后端、零上传——照片与轨迹始终只在你的浏览器里。

**在线使用：<https://hcazrej.github.io/trace-photo-overlay/>**

## 缘起

灵感来自 [旅图九格](http://43.136.132.103:3000/) 这个很有意思的小工具。我很喜欢它的点子，只是想要它能直接读我自己骑行 App 导出的轨迹文件，也希望照片只留在本地，于是照着这个想法写了一个纯前端版本——打开网页即用，不需要服务器，也不上传任何东西。

## 功能

- **九宫格拼图**：9 张照片可拖动排序，支持 jpg / png / webp / HEIC / 实况照片（均本地转换）
- **轨迹叠加**：把运动轨迹画到照片上，颜色、线宽可调
- **多格式 / 多文件**：支持 GPX / KML / TCX / FIT / GeoJSON / CSV；可一次导入多个文件按顺序首尾拼接（适合码表分段记录的多日长途）
- **数据卡**：标题 / 日期 + 距离 / 时长 / 均速 / 配速 / 爬升，从轨迹自动算出，也可手填
- **一键导出**：打包为 ZIP（`1.jpg`…`9.jpg`），按顺序发朋友圈即拼成九宫格

## 支持的轨迹格式

| 格式 | 类型 | 记录的信息 |
|---|---|---|
| GPX | 文本 | 带时间戳和海拔的经纬度点，最通用 |
| KML | 文本 | `gx:Track` / `LineString`，可含时间、海拔 |
| TCX | 文本 | 佳明格式，含经纬度、海拔、时间 |
| FIT | 二进制 | 佳明 / 顽鹿等码表格式，信息最全 |
| GeoJSON | 文本 | `LineString` / `MultiLineString`，常只有坐标 |
| CSV | 文本 | 每行 `lat,lng` 或 `lng,lat` |

## 数据与算法

距离、时长、均速、配速、总爬升都直接从轨迹文件计算：

- **距离** — 相邻点直线距离用 Haversine 公式累加
- **时长** — 运动时长，扣除速度低于 `0.5 m/s` 的停顿段
- **均速 / 配速** — 距离 ÷ 运动时长
- **总爬升** — 海拔先做移动平均平滑去噪，再累计上升超过阈值的部分

各家 App 的停顿判定与爬升口径不尽相同，导出文件也会损失气压计海拔等信息，因此算出的时长 / 爬升可能与你的 App 略有出入。距离最准，其余为合理估算——任何数字都能在界面里改成 App 显示的官方值。

## 隐私

代码全部开源，可自行审阅。运行时除按需加载一次 HEIC 解码库外没有任何网络请求，照片与轨迹不会离开你的设备。

## 开发

